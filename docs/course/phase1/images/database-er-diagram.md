【用户-角色-权限模型（RBAC）】

    ┌─────────┐       ┌─────────┐       ┌─────────┐
    │  用户    │◄─────►│  角色    │◄─────►│  权限    │
    │ (Admin) │  N:M  │ (Role)  │  N:M  │(Menu/   │
    └─────────┘       └─────────┘       │Permission)
                                          └─────────┘

【关系说明】
• 一个用户可以有多个角色（如：admin 既是系统管理员又是商品管理员）
• 一个角色可以分配给多个用户（如：多个用户都是商品管理员）
• 一个角色可以有多个菜单权限
• 一个角色可以有多个接口权限

【核心表】
• ums_admin：存储用户信息（用户名、密码、状态等）
• ums_role：存储角色信息（角色名称、描述等）
• ums_menu：存储菜单信息（菜单名称、图标、排序等）
• ums_permission：存储接口权限信息（URI、请求方式等）

【关系表】
• ums_admin_role_relation：用户与角色的关联
• ums_role_menu_relation：角色与菜单的关联
• ums_role_permission_relation：角色与权限的关联
